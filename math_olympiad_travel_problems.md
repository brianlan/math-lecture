# 小学奥数行程问题常见题型全解

行程问题是小学奥数中非常重要且类型丰富的一个专题。其核心公式为：**路程 = 速度 × 时间** ($s = v \times t$)。

以下是小学奥数中常见的行程问题题型分类及场景解析：

## 1. 基础行程问题 (Basic Travel Problems)
这是最基本的类型，通常只涉及一个运动主体。
*   **场景**：一个人或车从A地到B地。
*   **核心**：直接运用 $s=vt$ 及其变形公式。
*   **常见考点**：单位换算（如 m/s 到 km/h）、简单的往返运动。

## 2. 相遇问题 (Encounter Problems)
两个物体从两地出发，相向而行（面对面），最终在途中相遇。
*   **场景**：甲乙两人分别从A、B两地同时出发，相向而行。
*   **核心公式**：**路程和 = 速度和 × 相遇时间**
*   **进阶场景**：
    *   **二次相遇/多次相遇**：两人相遇后继续前行，到达对方起点后立即返回，再次相遇。
        *   *规律*：第二次相遇时，两人走的总路程是第一次相遇时总路程的3倍（从出发开始算）。
    *   **不同时出发**：一人先走，另一人后走，最终相遇。

## 3. 追及问题 (Pursuit Problems)
两个物体同向而行，速度快的在后面追速度慢的。
*   **场景**：甲在乙后面，甲的速度比乙快，甲追乙。
*   **核心公式**：**路程差 = 速度差 × 追及时间**
*   **常见考点**：
    *   判断能否追上。
    *   计算追上所需的时间或距离。
    *   **环形追及**：在环形跑道上，快者追上慢者一圈（路程差为一圈长度）。

## 4. 火车过桥/过人/错车 (Train Problems)
此类问题的特点是**物体本身有长度**，不能看作质点。
*   **场景**：
    *   **火车过桥/隧道**：从车头进桥到车尾离桥。
        *   *公式*：**总路程 = 桥长 + 车长**
    *   **火车过人**：
        *   人静止：相当于过一个长度为0的物体，路程 = 车长。
        *   人迎面走（相遇）：路程 = 车长，速度 = 车速 + 人速。
        *   人同向走（追及）：路程 = 车长，速度 = 车速 - 人速。
    *   **两车错车**：两列火车交错而过。
        *   *公式*：**错车时间 = (甲车长 + 乙车长) ÷ 速度和**
    *   **两车超车**：快车超过慢车。
        *   *公式*：**超车时间 = (甲车长 + 乙车长) ÷ 速度差**

## 5. 流水行船 (Boat in Current Problems)
船在流动的水中航行，速度受水流影响。
*   **场景**：船在河里顺流而下或逆流而上。
*   **核心概念**：
    *   **顺水速度 = 船速 + 水速**
    *   **逆水速度 = 船速 - 水速**
    *   *推论*：船速 = (顺水速度 + 逆水速度) ÷ 2；水速 = (顺水速度 - 逆水速度) ÷ 2
*   **常见考点**：
    *   **漂流瓶问题**：物体无动力漂流，速度等于水速。
    *   **水中失物 (Lost Object)**：船掉东西后返回追。
        *   *核心结论*：**掉头追及时间 = 掉落后继续行驶时间**（与水速无关，相当于在静水中掉东西回头追）。

## 6. 环形跑道 (Circular Track Problems)
在封闭的环形路线上运动。
*   **场景**：两人在操场跑道上跑步。
*   **核心类型**：
    *   **反向运动（相遇）**：每相遇一次，两人合走一圈。
        *   *公式*：**相遇时间 × 速度和 = 跑道周长**
    *   **同向运动（追及）**：每追上一次，快者比慢者多走一圈。
        *   *公式*：**追及时间 × 速度差 = 跑道周长**

## 7. 钟面行程 (Clock Problems)
将时钟的时针和分针看作在环形跑道上运动的两个物体。
*   **场景**：求某时刻时针和分针重合、垂直或成一定角度。
*   **核心数据**：
    *   **分针速度**：6度/分钟 （或 1格/分钟）
    *   **时针速度**：0.5度/分钟 （或 1/12格/分钟）
    *   **速度差**：5.5度/分钟 （或 11/12格/分钟）
*   **解题思路**：通常转化为**追及问题**处理。

## 8. 走走停停 (Stop-and-Go Problems)
运动过程中有休息或停止的情况。
*   **场景**：甲每走30分钟休息5分钟。
*   **解题技巧**：将“走+停”看作一个周期，计算总时间包含多少个周期，注意最后一段路程可能不需要休息。

## 9. 接送问题 (Pick-up and Drop-off Problems)
只有一辆车，需要接送多人或多批人到达目的地。
*   **场景**：一辆摩托车接送甲乙两人去某地，先送甲一段，放下甲让其步行，车回头接乙。
*   **核心思想**：**最优化原则**，通常要求所有人同时到达目的地（时间最短）。
*   **解题技巧**：利用画图法，结合比例关系（人速与车速之比）求解。

## 10. 电梯行程 (Escalator Problems)
人在自动扶梯上行走，类似流水行船，但通常涉及“级数”。
*   **场景**：人在向上运行的扶梯上向上走（顺流）或向下走（逆流）。
*   **核心公式**：
    *   **顺行**：(人速 + 电梯速) × 时间 = 可见级数
    *   **逆行**：(人速 - 电梯速) × 时间 = 可见级数
    *   *注意*：这里的“速度”通常指单位时间走的级数。

## 11. 猎狗追兔 (Dog Chasing Rabbit)
涉及步长（每步距离）和频率（单位时间步数）的差异。
*   **场景**：狗跑3步的时间兔子跑4步，狗跑2步的距离等于兔子跑3步的距离。
*   **解题技巧**：先统一单位，计算出**速度比**。
    *   速度 = 步长 × 频率。

## 12. 平均速度 (Average Speed)
*   **定义**：**平均速度 = 总路程 ÷ 总时间**
*   **易错点**：平均速度 $\neq$ 速度的平均值（(v1+v2)/2 是错误的，除非时间相同）。
*   **常见场景**：上山下山往返，求平均速度。

## 13. 变速问题 (Variable Speed Problems)
路程中速度发生变化。
*   **场景**：汽车在高速公路上速度是100，在普通公路上是60。
*   **解题技巧**：分段计算，或利用比例关系（路程一定，速度与时间成反比）。

## 14. 多人行程 (Multiple Objects)
涉及三个或更多人的运动关系。
*   **场景**：甲乙丙三人，甲乙相遇后多久甲遇到丙。
*   **解题技巧**：两两分析，转化为多个相遇或追及问题组合。

## 15. 十字路口与红绿灯 (Traffic Light Problems)
*   **场景**：通过一系列有红绿灯的路口。
*   **核心**：考虑等待时间，寻找周期规律。

## 16. 队伍行进 (Marching Column Problems)
*   **场景**：通讯员从队尾跑到队头再返回。
*   **核心**：
    *   从队尾到队头：追及问题（人追队头）。
    *   从队头到队尾：相遇问题（人迎队尾）。

## 17. 发车问题 (Departure Interval Problems)
涉及车辆按固定时间间隔发车的问题。
*   **场景**：某人沿公路行走，每隔5分钟有一辆公交车从后面追上（追及），每隔3分钟有一辆公交车迎面开来（相遇）。
*   **核心**：
    *   **发车间隔** (距离) = 速度 × 时间间隔
    *   **追及**：发车间隔 = (车速 - 人速) × 追及间隔时间
    *   **相遇**：发车间隔 = (车速 + 人速) × 相遇间隔时间
    *   *推论*：车速和人速可以通过这两个方程联立求解。

## 18. 往返跑/狗跑模型 (Back-and-Forth Running)
一个物体在两个运动物体之间不停往返。
*   **场景**：甲乙相向而行，一只狗从甲处跑到乙处，遇到乙立即跑回甲处，如此往返，直到甲乙相遇。
*   **核心思想**：**转化法**。
    *   狗跑的总时间 = 甲乙相遇的时间（因为狗一直在跑，直到两人相遇才停）。
    *   **狗跑总路程 = 狗速 × 甲乙相遇时间**。
    *   *注意*：不需要计算狗具体跑了多少个来回，直接用总时间计算。

## 19. 快慢钟问题 (Fast and Slow Clocks)
涉及走时不准的钟表，本质是**变速问题**或**比例问题**。
*   **场景**：标准时间过1小时，坏钟走了62分钟（快钟）或58分钟（慢钟）。
*   **核心**：建立“标准时间”与“坏钟时间”的比例关系。
    *   $\frac{\text{坏钟时间}}{\text{标准时间}} = \frac{\text{坏钟速度}}{\text{标准速度}}$
*   **解题技巧**：统一换算为标准时间进行比较，或利用“校对周期”求解。

## 20. S-T图象与柳卡图 (S-T Graphs & Willow Card)
利用距离-时间图象（S-T图）解决复杂的行程问题，特别是多人多次相遇。
*   **场景**：题目直接给出图象，或题目描述极其复杂（如多人多次往返），难以用公式直接列式。
*   **核心**：
    *   **斜率**：代表速度。
    *   **交点**：代表相遇或追及。
    *   **柳卡图 (Liu Ka Diagram)**：一种专门处理多人多次相遇问题的折线图法，通过寻找几何相似三角形关系求解。

# 核心数学模型总结 (The Essence)

尽管行程问题题型繁多（已扩展至20种），但它们都可以归纳为以下**三大核心数学模型**。掌握这些模型，就能看透题目表象，直击本质。

## 模型一：相对速度模型 (Relative Speed Model)
**本质**：通过改变参考系，将“两个物体的复杂运动”简化为“一个物体相对于另一个物体的简单运动”。

*   **适用场景**：
    *   **基础类**：相遇、追及、火车错车/超车。
    *   **介质类**：流水行船（顺/逆）、电梯行程（顺/逆）。
    *   **特殊类**：钟面行程（追及）、队伍行进（人追队/队迎人）、发车问题（车追人/车迎人）。
    *   **转化类**：往返跑/狗跑模型（关键在于利用相对速度求出“共同运动时间”）。
*   **核心公式**：
    $$ \Delta S = V_{\text{rel}} \times t $$
    其中 $\Delta S$ 是路程差（或初始距离），$V_{\text{rel}}$ 是相对速度。
*   **两种形态**：
    1.  **相向/背向运动（合作模型）**：$V_{\text{rel}} = V_1 + V_2$
    2.  **同向运动（竞争模型）**：$V_{\text{rel}} = |V_1 - V_2|$

## 模型二：比例与几何模型 (Proportionality & Geometric Model)
**本质**：利用速度、时间、路程三者之间的正反比关系，或利用图象中的几何性质（相似三角形、斜率）解题。

*   **适用场景**：
    *   **比例类**：接送问题、猎狗追兔（步长/频率比）、变速问题、快慢钟问题（标准与坏钟的比）。
    *   **几何类**：S-T图象、柳卡图（利用图象交点和相似三角形解决多人多次相遇）。
*   **核心关系**：
    1.  **时间 $t$ 相同**：$S_1 : S_2 = V_1 : V_2$ （柳卡图中常利用沙漏模型/金字塔模型推导此关系）。
    2.  **路程 $s$ 相同**：$t_1 : t_2 = V_2 : V_1$ （快慢钟、往返运动常用）。
    3.  **速度 $v$ 相同**：$S_1 : S_2 = t_1 : t_2$。

## 模型三：周期与分段模型 (Cycle & Segment Model)
**本质**：将连续的复杂运动分解为若干个重复的“周期”或独立的“片段”。

*   **适用场景**：
    *   **周期类**：走走停停、环形跑道（多次相遇/追及）、十字路口红绿灯、发车问题（时间间隔）。
    *   **分段类**：平均速度（总路程/总时间）、多段变速运动。
*   **核心思想**：
    1.  **周期法**：找到最小循环单元（Cycle），利用 $\text{Total} = k \times \text{Cycle} + \text{Remainder}$ 求解。
    2.  **分段法**：将全程分为几段（如上坡、平路、下坡），每段单独计算，最后汇总。

## 高阶辅助方法
当题目涉及多角色调度、概率期望或极其复杂的多人往返时，可以在三大核心模型之上叠加以下三种工具，帮助理清事件顺序与约束。

### 1. 事件时间线 + 状态图 (Event Timeline with States)
将“谁在做什么”映射到一条时间轴，并标记关键事件（接人、放人、停靠、等待等）和状态（载客/空车/步行）。

```
时间 →
┌──────────┬───────┬────────────┬───────┐
│ 载客前往 │ 空车返回 │ 载客+等待 │ 再次前往 │  ……  (车辆 A)
└──────────┴───────┴────────────┴───────┘
        ↑t1         ↑t2            ↑t3
```

* **做法**：令 \(t_1,t_2,\ldots\) 表示事件时刻；在每段状态上运用相对速度或分段模型，写出“速度 × 时间 = 距离”以及“同时到达/同步”的等式。
* **适用**：接送问题（题 176–184、221）、需要安排会合顺序的多段行程、必须满足“所有人同时到达”的调度题。

### 2. 概率与期望模型 (Probabilistic Cycle Model)
在周期模型基础上，考虑“随机到达”或“期望等待时间”。核心想法是把随机时刻映射到周期区间，对每个区间求等待函数并取平均。

```
周期 T = 红灯(90s) + 绿灯(60s)
等待时间 w(t) = 
    ├ 0, 若 t 落在绿灯段
    └ 90 - t, 若 t 落在红灯段
期望等待  E[w] = (1/T) ∫_0^T w(t) dt
```

* **做法**：将“随机到达”视为在一个周期内均匀分布；写出等待函数 \(w(t)\)，在周期内积分或求平均。
* **适用**：题 191 等需要期望等待/排队时间的红绿灯、轮渡、发车频率问题。

### 3. 多主体事件图 (Multi-entity Event Graph)
将每个运动主体画成一条距离-时间折线（类似扩展的 S-T 图），用交点表示相遇、用水平段表示等待、用斜率表示速度。适合处理有多只狗、无人机、车辆等同时运动且互相依赖的题目。

```
距离
│        /甲车
│   ___ /¯¯¯¯¯¯¯¯
│  /   /        ↘ 无人机
│ /乙车          \
└───────────────── 时间
```

* **做法**：为每个对象绘制折线，交点代表事件；利用斜率得到速度关系，再结合相对速度与周期模型求各段时间/距离。
* **适用**：多狗/无人机往返（题 195–202, 222）、多人多次相遇并含停顿的复杂题、柳卡图的推广。

掌握上述高阶辅助方法，可以在核心三模型的基础上处理复杂调度、期望值以及多实体联动，确保面对更“花”题目时也能落到可计算的方程或图象分析之中。
